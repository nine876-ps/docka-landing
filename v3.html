What is a runbook?

01

One task per runbook

Each runbook covers exactly one task. Narrow scope means agents execute without ambiguity.

02

Written for agents, not humans

Explicit auth steps, exact API calls, enum values, error patterns, and a machine-verifiable success signal.

03

Verified with real agents

Tested end-to-end with ChatGPT, Claude, and Gemini. If any agent fails, the runbook doesn't ship.

04

Published and indexed for agent web search

Hosted on your subdomain, structured for agent queries — not for Google, for agents.
