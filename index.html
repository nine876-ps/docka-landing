FAQ

Common questions

Everything you need to know before booking a demo.

How is Docka different from Kapa.ai or RAGAS? ⌄
Different layer entirely. Kapa measures how helpful an AI chatbot is when users ask questions. RAGAS measures whether your LLM retrieves context well. Docka measures whether real agents can complete real tasks using your docs — install, auth, first API call. Non-overlapping, and we integrate with both.
What does "mechanical scoring" mean? ⌄
We don't use an LLM to judge whether the agent's answer was good. We diff actual CLI output, HTTP response codes, and SDK behavior against known-good baselines. If xtrabackup --prepare exits 0 with the expected output — pass. If the agent called --apply-log and got a 127 — fail, minus 15. No subjectivity, no prompt drift.
Do I need to change my docs before using Docka? ⌄
No. We crawl your live docs as-is. The whole point is to find what's already broken before your users' agents do. We run, we score, we give you the fix list. You change the docs. We re-run and confirm they pass.
How do you generate the agent tasks? ⌄
We focus on revenue-critical paths: the 5–15 tasks that drive 80% of integration outcomes — install, authenticate, create a resource, configure the most common integration, handle errors. Tasks are hand-curated for each pilot based on your docs and verified by you before anything runs.
Can this run in CI/CD? ⌄
Yes — that's the SaaS subscription tier. Docka runs on every doc publish, catches regressions before they go live, and can block merges that would drop your Agent-Readiness Score below a threshold you set. Same output format as the audit, automated.
How is this priced? ⌄
Two tiers: Per-audit ($5–25K, one-shot) for discovery and first-pass evaluation. SaaS subscription ($1–5K/month) for continuous CI integration and ongoing monitoring. We start every engagement with an audit — contact us for a quote specific to your docs size and complexity.
Is there an "Agent-Readiness Score" definition or standard? ⌄
The Agent-Readiness Score is a 0–100 metric measuring how reliably AI agents complete revenue-path tasks using your documentation. Broken down by task, failure category (deprecated call, wrong endpoint, renamed flag, missing context), and severity. We publish the methodology openly and are working toward an industry benchmark via the annual State of API-first Documentation report.
What if my docs are private or behind auth? ⌄
We can work with exported snapshots or staging environments. We don't need production access — we just need the same content an agent would see. For private docs, we set up a simple crawl token during onboarding.
