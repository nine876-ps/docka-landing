ARS Framework — Agent Readiness Score

Score = (DQ × 0.30) + (ADE × 0.45) + (HIR × 0.25). Higher score = harder for agents to complete tasks autonomously.

DQ Docs Quality 30%

Accuracy, deprecated calls, example correctness, schema consistency. Does the docs match what the API actually does?

ADE API Design Efficiency 45%

Tokens-to-task, mandatory discovery calls, batch support, error quality. How many calls to complete a task?

HIR Human Interruption Rate 25%

Setup friction, auth discoverability, flow blockers, self-recovery. How many times must a human intervene?

Three Failure Archetypes

Category A

Architecturally Incompatible

No headless path exists. The product's core value is locked behind a UI — agents can only trigger webhooks or call enterprise-gated APIs.

Canva (91) Clay (89) ZoomInfo (88)

Category B

Token Amplifiers

3–5× token waste from mandatory discovery calls before useful work. APIs are functional but require many prerequisite lookups.

Attio (65) Lusha (60) Snov.io (68) Outreach.io (80)

Category C

Silent Success / Wrong Data

HTTP 200 returned with empty or incorrect data. The agent thinks it succeeded. The user sees nothing. The worst failure mode.

Apollo (55) Amplitude (70) Jira (77) Linear (46) Smartlead (63) Lusha (60)

Key insight: Absence of API is not a death sentence. Clay has no REST API — but agents can push events via webhook, Clay runs the enrichment waterfall, and outreach fires automatically. Canva's API can't edit content — but "generate 50 banner variants for each ICP segment" is a real request agents should handle. The question is never "is the API ready?" — it's "would a user ask an agent to do this?"

CEO / Founder

  • Developer community posts: "tried to use [product] with Claude — doesn't work"
  • Being on the wrong side of the agentic platform transition
"I don't want to be the product that breaks every AI workflow. That's a reputation we can't afford in 2026."
Buys for
Competitive moat · Developer retention · Category positioning as "agent-native"

CPO / Head of Product
Owns developer experience, gets DevRel escalations about agent failures
What keeps them up at night
  • Support tickets: "Claude can't authenticate to your API", "error messages are unparseable"
  • No metric to prioritize DX improvements on the roadmap
  • Shipping a breaking change that silently breaks all agent workflows
"I know our docs need work but I can't prioritize it without data. Docka gives me the data — and a score I can track sprint over sprint."
Buys for
ARS as a KPI · Regression detection in CI/CD · Roadmap prioritization signal
📈
Head of Sales / RevOps
Building AI-powered outreach — agents managing the full sequence lifecycle
What keeps them up at night
  • 20+ back-and-forths with Claude to set up one Outreach.io campaign
  • Agents fail silently — campaign "created" but no emails sent (Smartlead Cat C)
  • Stack of tools that each require human babysitting: Clay → Instantly → CRM sync
"If my agent can set up and manage campaigns without me, I can run 10× more experiments. The ROI is obvious — but right now every tool in my stack requires hand-holding."
Buys for

⚙️
CTO / Head of Engineering
Technical owner of API contract, CI/CD, and agent infrastructure
